# 工場見学用リアルタイム音声配信WEBアプリ システム要件

## 概要
説明者（1人）がスマホのイヤホンマイクで話した音声を、
聴講者（10〜30人）のスマホブラウザにリアルタイムで配信するシステム。
アプリのインストールは不要で、ブラウザだけで動作することが条件。

## 技術要件
- 通信方式：WebSocket（音声チャンクをリアルタイム中継）
- 配信側：MediaRecorder API でマイク音声を取得
- 受信側：Web Audio API で音声を再生
- サーバー：Node.js + ws ライブラリ
- HTTPS必須（マイクアクセスのためブラウザが要求）
- ネットワーク：外部インターネット経由（VPS上で動作）

## 作成するファイル
1. **server.js** ：Node.js WebSocketサーバー（音声中継）
2. **host.html** ：説明者用ページ（マイク取得・配信開始/停止ボタン）
3. **listen.html**：聴講者用ページ（自動受信・再生、接続人数表示）
4. **README.md** ：VPSへのセットアップ手順（Node.jsインストール〜SSL取得〜起動まで）

## 各ページの仕様

### host.html（説明者用）
- マイクの使用許可をリクエスト
- 「配信開始」「配信停止」ボタン
- 現在の接続人数をリアルタイム表示
- 配信中であることがわかるインジケーター表示

### listen.html（聴講者用）
- ページを開くと自動的に受信待機
- 配信が始まると自動再生
- 接続状態（待機中・受信中）を表示
- QRコードでURLを共有しやすい設計（URLはシンプルに）

### server.js
- 説明者と聴講者を区別して管理
- 説明者から受け取った音声チャンクを全聴講者にブロードキャスト
- 接続人数を全クライアントに通知
- 説明者が切断したら聴講者に通知

## 非機能要件
- 遅延：2秒以内を目標
- 同時接続：最大30人
- 複数の見学グループが同時進行する可能性があるため、
  ルーム機能（URLパラメータで部屋を分ける 例：?room=A）があると望ましい

## 本番セットアップ手順（README）に含めること
- Node.jsのインストール
- 必要パッケージのインストール（npm install）
- Let's EncryptによるSSL証明書の取得方法
- サーバー起動コマンド
- 聴講者へのQRコード配布方法のヒント

## ローカルテスト環境
- PCのlocalhost上でサーバーを動かしてテストしたい
- スマホ実機テストにはngrokを使う想定
  （理由：LAN内のスマホから http://192.168.x.x でアクセスするとHTTPSでないため
  ブラウザがマイクの使用を許可しないため）
- READMEに以下のローカルテスト手順を含めること
  1. Node.jsでサーバーをローカル起動する方法
  2. PCブラウザで host.html を開いて単体確認する方法
  3. ngrokのインストールと起動方法
  4. ngrokで発行されたHTTPS URLをスマホで開いて listen.html をテストする方法
